シーケンス図の開始と終了

シーケンス図全体を開始するには、@startumlと@endumlを使用します。これらのキーワードで図の始まりと終わりを示します。

    
@startuml
... // シーケンス図の内容
@enduml

参加者（オブジェクト）の定義

シーケンス図に登場するオブジェクト（参加者）を定義します。participantキーワードを使用します。オブジェクト名を<<>>で囲むことで、オブジェクトの種類を指定することもできます。

            
@startuml
participant "Actor 1" as Actor1
participant "Actor 2" as Actor2 << Database >>
@enduml

メッセージの送信

オブジェクト間でのメッセージの送信を示します。矢印を使ってメッセージの流れを示し、->または --> で送信元から送信先を指定します。

            
@startuml
participant "Actor 1" as Actor1
participant "Actor 2" as Actor2 << Database >>

Actor1 -> Actor2: メッセージ内容
@enduml

メッセージの応答

オブジェクトがメッセージに応答する場合、応答メッセージを描くことができます。-->または<--を使用して、応答の方向を示します。

            
@startuml
participant "Actor 1" as Actor1
participant "Actor 2" as Actor2 << Database >>

Actor1 -> Actor2: メッセージ内容
Actor2 --> Actor1: 応答内容
@enduml

ループや条件分岐

シーケンス図にはループや条件分岐も描くことができます。altとelseを使って条件分岐を、loopを使ってループを示します。

            
@startuml
participant "Actor 1" as Actor1
participant "Actor 2" as Actor2 << Database >>

Actor1 -> Actor2: メッセージ内容
Actor2 --> Actor1: 応答内容

alt 条件A
    Actor1 -> Actor2: メッセージA
else 条件B
    Actor1 -> Actor2: メッセージB
end

loop 5 times
    Actor1 -> Actor2: メッセージ
end
@enduml

コメント

シーケンス図にコメントを追加するには、note left ofやnote right ofを使って、オブジェクトの横にコメントを配置します。

            
@startuml
participant "Actor 1" as Actor1
participant "Actor 2" as Actor2 << Database >>

Actor1 -> Actor2: メッセージ内容
Actor2 --> Actor1: 応答内容

alt 条件A
    Actor1 -> Actor2: メッセージA
else 条件B
    Actor1 -> Actor2: メッセージB
end

loop 5 times
    Actor1 -> Actor2: メッセージ
end

note left of Actor1
    これはコメントです
end note

@enduml

シーケンス図の構文と機能の詳細はPlantUMLの公式ドキュメンテーションを参照してください。
