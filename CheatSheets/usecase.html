ユースケース図の開始と終了

ユースケース図全体を開始するには、@startumlと@endumlを使用します。これらのキーワードで図の始まりと終わりを示します。

    
@startuml
... // ユースケース図の内容
@enduml

アクターの定義

アクターはユーザーや外部エンティティを表します。actorキーワードを使用してアクターを定義します。

            
@startuml
actor User
actor "外部システム" as ExternalSystem
@enduml

ユースケースの定義

ユースケースはシステムの機能や機能を表します。usecaseキーワードを使用するか、単語を丸かっこで囲んでユースケースを定義します。

            
@startuml
actor User
actor "外部システム" as ExternalSystem

usecase ユーザー登録
(ログイン)
(注文作成)
@enduml

アクターとユースケースの関連付け

アクターとユースケースを関連付け、アクターがどのユースケースに参加するかを示します。

            
@startuml
actor User
actor 外部システム as ExternalSystem

usecase ユーザー登録
(ログイン)
(注文作成)

User --> ユーザー登録
User --> ログイン
ExternalSystem --> 注文作成
@enduml

アクターとユースケースの関連詳細

関連に詳細情報を追加することができます。例えば、アクターが特定のユースケースを実行する際の条件や制約などを記述できます。

            
@startuml
actor User
actor 外部システム as ExternalSystem

usecase ユーザー登録
(ログイン)
(注文作成)

User --> ユーザー登録
User --> ログイン
ExternalSystem --> 注文作成

User --> ユーザー登録 : ユーザー情報を提供
ExternalSystem --> 注文作成 : 注文情報を受信
@enduml

アクターやユースケースをグループ化

packageキーワード、rectangleキーワードを使用して、アクターやユースケースをグループ化できます。

            
@startuml
actor User
actor 外部システム as ExternalSystem

package システム {
    rectangle ユーザー管理 as UserManager {
    usecase ユーザー登録
    (ログイン)
    }

rectangle 注文処理 as OrderProcessing {
    (注文作成)
    }
}

User --> ユーザー登録
User --> ログイン
ExternalSystem --> 注文作成

User --> ユーザー登録 : ユーザー情報を提供
ExternalSystem --> 注文作成 : 注文情報を受信
@enduml
