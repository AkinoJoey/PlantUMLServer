クラス図の開始と終了

クラス図全体を開始するには、@startumlと@endumlを使用します。これらのキーワードで図の始まりと終わりを示します。

    
@startuml
... // クラス図の内容
@enduml

クラスの定義

クラスとその属性、メソッドを定義します。クラス名はクラスボックス内に配置され、属性とメソッドはクラスボックス内にリスト形式で配置されます。

            
@startuml
class Post {
    -postID: int
    -content: string
    -user: User
    -comments: List< Comment >
    -likes: int
    +Post(postID: int, content: string, user: User)
    +getPostID(): int
    +getContent(): string
    +getUser(): User
    +addComment(comment: Comment): void
    +getComments(): List< Comment >
    +like(): void
    +getLikes(): int
}
@enduml

クラスの関連性の定義

クラス間の関連性を表現するために、関連線（Association Line）を使用します。-- は通常の関連性を示し、o-- は Extension を、*-- は Aggregation を、o--* は Composition を示します。

            
@startuml
class Post {
    -postID: int
    -content: string
    -user: User
    -comments: List< Comment >
    -likes: int
    +Post(postID: int, content: string, user: User)
    +getPostID(): int
    +getContent(): string
    +getUser(): User
    +addComment(comment: Comment): void
    +getComments(): List< Comment >
    +like(): void
    +getLikes(): int
}

class Comment {
}

class Like {
}

Post *-- Comment : Aggregation
Post *-- Like : Aggregation
@enduml

抽象クラス、インターフェースの定義

抽象クラスは abstract キーワードを使用して、クラスボックスの右上に書かれます。インタフェースはクラス図内でクラス名の下に << interface >> と表示されます。

            
@startuml
abstract class User {
    -userID: int
    -username: string
    -email: string
    +User(userID: int, username: string, email: string)
    +getUserID(): int
    +getUsername(): string
    +getEmail(): string
}
@enduml

静的メンバー、クラス変数

{static} または {abstract} 修飾子を使用して、静的または抽象的なメソッドを定義します。

            
@startuml
abstract class User {
    -userID: int
    -username: string
    -email: string
    +User(userID: int, username: string, email: string)
    +getUserID(): int
    +getUsername(): string
    +getEmail(): string
    {abstract} post(content: string): void
}
@enduml

多重度の定義

多重度を示すために、1, 0..1, *, 1..* などの数字を使用します。これらの数字は関連線の両側に配置されます。

            
@startuml
class Post {
    -postID: int
    -content: string
    -user: User
    -comments: List< Comment >
    -likes: int
    +Post(postID: int, content: string, user: User)
    +getPostID(): int
    +getContent(): string
    +getUser(): User
    +addComment(comment: Comment): void
    +getComments(): List< Comment >
    +like(): void
    +getLikes(): int
}

class Comment {
}

class Like {
}

Post *-- "*" Comment : Aggregation
Post *-- Like : Aggregation
@enduml
